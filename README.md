# Tokenised LLM — a feasibility course

A short, honest course exploring one question: **can you build an LLM that rewards the
people who contributed knowledge whenever someone else's query uses it?**

It works through the feasibility of the idea — where the hard problems really sit, and what
the honest trade-offs are — rather than hand-waving the hype.

## Lessons

1. **The hard part isn't the tokens** — the Bake ↔ Retrieve spectrum, and why *attribution*
   (not payment) decides whether the idea is buildable.
2. **From chunk to payout** — the money path end to end, an interactive payout-split
   calculator, and the gap that decides whether it's honest: *retrieved ≠ used*.

More lessons added over time.
